这个博客搭建用的是Hexo4.2 + Next7.7.1版本，适用于学过一点CSS的

安装Hexo流程

这一步其实就是按照Hexo文档安装就好了（Hexo文档）

注意Hexo的安装需要预先安装好Node.js，Git（点击进入官网下载，里面有详细的安装教程）还有npm（npm是随同NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题）

我电脑是Mac OS 15版本，因此需要添加sudo

安装成功后有两种办法调用Hexo

第二种办法需要将 Hexo 所在的目录下的 node_modules 添加到环境变量之中。对于Mac系统的用户而言，会自动添加到环境变量中，因此可以直接选择第二种，更加方便的

Hexo常用命令

  1. hexo init [folder]

    Hexo 将会在指定文件夹中新建所需要的文件

  2. hexo g或hexo generate

    该命令执行后在hexo站点根目录下生成public文件夹

  3. hexo clean

    将根目录下的public文件夹删除

  4. hexo s或hexo serve

    启动服务预览，通常是端口4000即http://localhost:4000/，Mac上暂停服务为^z

  5. hexo d或hexo deploy

    部署站点，在本地生成.deploy_git文件夹，并将编译后的文件上传至 GitHub

  6. hexo new [layout] <title>

    上述指令执行时，Hexo 会尝试在 scaffolds 中寻找layout.md（相对应的名字）布局，若找到，则根据该布局新建文章；若未找到或指令中未指定该参数，则使用post.md新建文章。新建文章的名称在_config.yml中配置。title则是文件名（可以使用双引号表示，也可以不用，中英即可）

  7. 更多的指令请参考 Hexo文档中的指令

Hexo + Github Page 搭建

根据上面教程完成步骤，应该就可以通过 hexo g && hexo s看到所搭建的博客，但是这个博客只是在本地上，并不能上传到网上，但通过GitHub Page，我们就可以将我们的博客放上网上了。

首先我们需要注册GitHub的账号

接着在GitHub个人主页上的十字Button中点击New Repository，这就是在GitHub创建一个代码仓库，用于存放相关代码。而这个Repository的名字应该是你的GitHub名+.github.io，也就是GitHub名为xxx，而这个Repository的名应该为xxx.github.io

紧跟着，找到Hexo根目录下的_config.yml文件，进去拉到最下面会看到以下代码（以刚才创建的xxxGitHub账号），type改为git，repo改为刚才创建的Repository所在网址

完成之后便可以使用 hexo deploy命令，将Hexo文件夹推到GitHub库中了。第一次使用的时候，可能需要输入GitHub的账号和密码（在命令行中），之后再次推送的时候，就使用 hexo clean && hexo deploy命令就行了。

Hexo主题更改

我使用的是Next7.7.1版本，如果不是使用Next主题，请忽视以下的内容，如果是使用Next小于7.3以及以下版本，可以直接在网上找大佬博客

更改主题

在Hexo官网主题中搜索Next下载（目前版本为Next7.7.1），将下载好的Next压缩包（我是直接下载zip的，也可以直接用命令 git clone https://github.com/theme-next/hexo-theme-next.git）放在Hexo根目录的theme文件夹中，然后在根目录的_config.yml文件中搜索 theme，将 landscape改为刚才下好的Next文件夹名字（Hexo默认theme就是landscape）

如果使用的是最新Hexo版本，使用Next7.3以下是不适用的，图标会显示不出，出现Html代码（可能是我Mac的原因？）
